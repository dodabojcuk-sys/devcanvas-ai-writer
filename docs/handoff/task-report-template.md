# Codex Task Execution Report Template

Every Codex task PR must create a task report at:

`docs/handoff/task-XXX.md`

The PR body must include a link to that task report.

## 1. Task Summary
Describe what this task changed.

## 2. Files Changed
List every changed file.

## 3. Kernel Impact
State `yes` or `no`, then explain briefly.

## 4. Runtime Impact
State `yes` or `no`, then explain briefly.

## 5. System Impact
State `yes` or `no`, then explain briefly.

## 6. UI Impact
State `yes` or `no`, then explain briefly.

## 7. Risk Level
State `low`, `medium`, or `high`.

## 8. Execution Notes
Describe how Codex executed the task and any scope constraints observed.

## 9. Validation
State whether tests, typecheck, build, connector checks, or manual review were completed.
